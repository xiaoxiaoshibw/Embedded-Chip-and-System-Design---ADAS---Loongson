CPU 亲和性调度机制

通过 CPUAffinity 将 ADAS 核心控制线程绑定到固定 CPU 核心，
减少跨核迁移带来的调度抖动和缓存失效，
提升 100 Hz 控制闭环的实时性和稳定性。



### ADAS 至少给 2 个核，gateway 单独 1 个核，监控/日志再单独 1 个核。

core0  控制主循环（干净实时核，独占）
core1  DDS + ML(onnx异步) + 串口 + 遥测 + 日志 + 心跳
core2  HIL 时=gateway / 平时空
core3  headroom 空

命令可以这样写：

启动 ADAS，绑定到 core0 和 core1：

```
taskset -c 0,1 python3 ADAS.py
```

启动 gateway，绑定到 core2：

```
taskset -c 2 python3 gateway.py
```

启动监控或 KPI 采集，绑定到 core3：

```
taskset -c 3 python3 edge_result_collector.py
```

 HIL 启动脚本已经支持 `ADAS_CPU_LIST`，可以这样：

```
ADAS_CPU_LIST=0,1 python3 start_hil_adas.py
```

gateway：

```
taskset -c 2 python3 gateway.py
```

监控：

```
taskset -c 3 python3 edge_result_collector.py
```

因为是 systemd 服务，`adas-node.service` 里加：

```
[Service]
CPUAffinity=0 1
```

gateway 服务里加：

```
[Service]
CPUAffinity=2
```

监控服务里加：

```
[Service]
CPUAffinity=3
```

然后重载 systemd：

```
sudo systemctl daemon-reload
sudo systemctl restart adas-node.service
sudo systemctl restart gateway.service
```
