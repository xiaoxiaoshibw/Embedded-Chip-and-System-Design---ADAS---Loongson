# 成果

竞赛成果产物（对比图表）与其生成工具。**工具类代码按用途分类存放**，图片单独成目录。

## 目录结构（分类原则）

```
成果/
├── 图片/                 # 最终对比图（PNG，交付用）
├── 数据/                 # 实测逐帧数据（CSV，图表的数据来源）
└── 工具/                 # 工具类代码，按用途分类
    ├── 数据采集/          # 从 headless CARLA 采集实测数据 → 写 成果/数据/*.csv
    │   └── capture_plot_data.py
    └── 绘图/              # 读 成果/数据/*.csv → 出 成果/图片/*.png
        └── plot_results.py
```

> 约定：以后新增任何工具/脚本，都按用途放进对应分类子目录（数据采集 / 绘图 / 分析 / …），
> 不要散落在根目录。

## 如何复现

```powershell
# 1) 先启动 CARLA（无显卡用 -RenderOffScreen）
cd D:\Code\自动辅助驾驶仿真平台\CALRA
.\CarlaUE4.exe -RenderOffScreen -quality-level=Low

# 2) 采集实测数据（写 成果/数据/*.csv）
python 成果\工具\数据采集\capture_plot_data.py

# 3) 出图（写 成果/图片/*.png）
python 成果\工具\绘图\plot_results.py
```

“无创新基线”= 在同一套控制栈上关闭创新点（接近调速器 `ACC_APPROACH_DECEL`、
受控接近门控 `AEB_REQUIRED_DECEL_TRIGGER`、ML 预警）后的对照，**非另写一套弱系统**，
对比公平。速度边界 CSV 来自本机限速器 ON 的扫描实测。

> 注：无显卡机器上 CARLA 服务端每次 `load_world` 后易崩，采集脚本一次 `load_world`
> 跑完全部 5 段；若中途崩溃，重启 CARLA 再跑一次即可。
