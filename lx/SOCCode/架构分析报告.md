## 一、系统概览

### 1.1 系统定位

本系统是一个 **ROS2 单节点 ADAS 控制程序**，运行在一对 Jetson Nano 开发板上（一主一备），实现以下功能：

| 功能 | 缩写 | 说明 |
|------|------|------|
| 车道保持辅助 | LKA | 通过航向 PID + CTE 修正 + 曲率前馈计算方向盘转角 |
| 自适应巡航控制 | ACC | 距离误差 + 积分 + 速度差 + 前车加速度前馈的纵向控制器 |
| 自动紧急制动 | AEB | TTC/距离触发的紧急制动叠加 |
| 弯道保持 | — | 弯道中丢失前车时 PI 速度保持 |
| 超车决策 | — | 检测前方静止车辆后变道超车的五状态机 |
| 多目标跟踪 | — | 多候选车辆前车选择 + cut-in 预判 |

**通信架构**：

```
感知层(Simulink/ROS2) ──话题──→ Jetson Nano(本系统) ──串口──→ ESP32(执行器)
                                                  ↕
                                              UDP心跳
                                                  ↕
                                         Jetson Nano(备机)
```

### 1.2 核心约束

| 约束 | 值 | 影响 |
|------|-----|------|
| Python 版本 | 3.6（JetPack 4.x） | 不允许 PEP 604 联合类型、walrus 运算符、match 语句 |
| 控制频率 | 100 Hz (10ms) | 所有阻塞 I/O 必须在后台线程 |
| 串口波特率 | 115200 | ESP32 帧必须短小精悍 |
| 心跳超时 | 0.5s | 5 个周期无心跳即判主机失联 |
| 接管保护窗口 | 可配（`TAKEOVER_GUARD_DURATION_S`） | 备机接管后约束变化率 |

```
    │LaneWidthEst  │ │LonSmoothing   │ │  各算法模块      │

    └──────────────┘ └───────────────┘ └────────────────┘
```

**关键依赖规则**：

- `config.py` 被所有模块星导入（`from config import *`），是唯一的参数来源
- `runtime.py` 必须在 `ADAS.main()` 开头通过 `configure_runtime()` 配置后才能使用，其他模块只能 `import runtime` 读属性，不能 `from runtime import IS_PRIMARY`（会拿到未配置的默认值）
- `pipeline.py` 是纯计算入口，不依赖 ROS/串口/心跳，可被 `replay.py` 和 `run_scenario.py` 离线驱动
- `control/` 子包的算法模块互相通过数据类（`state.py`/`context.py`）传参，不直接引用 `ADAS.py`

## 三、数据模型

### 3.1 核心数据流

```
ROS话题回调                       控制管线                           输出
────────────                     ──────────                        ─────
/car1_xy  ─┐                     
/car1_psi ─┤                     
/car1_v   ─┤                     
/car2xy   ─┤→ VehicleSignals ──→ run_pure_pipeline() ──→ LateralContext
/car2_v   ─┤   (每拍由回调     (与在线等价的        ──→ LeadContext
/road_psi ─┤    写入，控制      纯计算管线)          ──→ LongitudinalContext
/heng_error─┘    环只读)                              ──→ lon_cmd
                                                     │
                                                     ├──→ ROS 发布 (8话题)
                                                     ├──→ ESP32 串口帧
                                                     └──→ UDP 心跳广播
```

### 3.2 VehicleSignals（感知输入，回调写入）

| 字段 | 类型 | 来源 | 说明 |
|------|------|------|------|
| `ego_x/y/yaw` | float | /car1_xy, /car1_psi | 自车位姿与航向 |
| `ego_v` | float | /car1_v | 自车速度 |
| `lead_x/y/yaw` | float | /car2xy | 前车位姿与航向 |
| `lead_v` | float | /car2_v | 前车速度 |
| `road_psi` | float | /road_psi | 道路航向 |
| `lane_offset` | float | /heng_error | 车道横向偏移 |
| `*_received` | bool | — | 各话题是否已收到 |
| `*_last_rx` | float | — | 各话题最近接收时刻（卡帧检测） |

### 3.3 ControlMemory（跨周期有状态）

| 字段 | 类型 | 说明 |
|------|------|------|
| `dt` | float | 控制周期（秒），从 LOOP_HZ 推导 |
| `cycle_count` | int | 周期计数器 |
| `psi_i_term` | float | 航向PID积分项累积 |
| `filtered_road_psi` | float | 低通滤波后的道路航向 |
| `filtered_cte` | float | 低通滤波后横向误差 |
| `filtered_curv` | float | 低通滤波后曲率 |
| `last_delta` | float | 上一拍方向盘转角（变化率限幅基准） |
| `target_lane_offset` | float | 超车目标偏移（双车道） |
| `filtered_lead_v_proj` | float | 滤波后前车投影速度 |
| `filtered_lead_accel` | float | 滤波后前车加速度 |
| `lane_safe/warn/hard_margin` | float | 车道余量（由LaneWidthEstimator刷新） |

## 四、控制管线详解

### 4.1 单周期执行流程（100Hz）

每个 10ms 周期，`_control_loop_impl()` 依次执行以下步骤：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    单周期控制流程 (10ms)                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ① esp32.drain_rx()                                                │
│     └── 无操作（串口读在rx线程，这里仅为接口兼容）                      │
│                                                                     │
│  ② peer_hb.is_active() → peer_active                               │
│     └── 主机：恒True；备机：超时/SEQ停滞→False                      │
│                                                                     │
│  ③ evaluate_control_health() → ControlHealth                       │
│     └── 检查: peer_active / ego_ready / road_ready / lead_ready    │
│     └── ego/road 卡帧检测: now - last_rx > SENSOR_STALE_TIMEOUT_S  │
│                                                                     │
│  ④ 门控判断                                                         │
│     ├── !peer_active → send_backup_alive(False); return            │
│     └── !(ego_ready && road_ready) → 轻制动; return                 │
│                                                                     │
│  ⑤ _handle_takeover_edge(now)                                      │
│     └── 备机接管边沿: 从主机最后帧种子初始化lon_smooth/last_delta    │
│     └── 启动 TAKEOVER_GUARD 窗口（更严变化率限幅）                   │
│                                                                     │
│  ⑥ _select_primary_lead(now)                                       │
│     └── 多目标跟踪器选主前车，写回signals.lead_*                    │
│                                                                     │
│  ⑦ run_pure_pipeline() ─────→ ★★★ 核心计算 ★★★                   │
│     ├── 超车状态机.update() → target_lane_offset                    │
│     ├── LaneWidthEstimator.update() → cur_lane_width + margins      │
│     ├── compute_lateral_command() → LateralContext                  │
│     │   ├── 道路航向低通滤波 → filtered_road_psi, rrate            │
│     │   ├── 预览航向 / 曲率 / 转向率计算                            │
│     │   ├── 航向PID: Δpsi = K_P*I*I + K_P*P*err + K_P*D*d_err     │
│     │   ├── CTE修正: Δcte = K_CTE * filtered_cte                   │
│     │   ├── 曲率前馈: Δff = K_FF * curv * (L + dyn_preview * v)   │
│     │   ├── 边界修正: 软约束 + 硬约束 + 制动                        │
│     │   ├── delta_rate_limit → 最终delta                            │
│     │   └── 边界修正独立于道路帧门控，按最新lane_offset刷新          │
│     ├── LeadTracker.evaluate() → LeadContext                        │
│     │   ├── 全局→自车坐标投影 + 低通滤波                            │
│     │   ├── 动态横向窗口（曲率自适应）                                │
│     │   ├── 多级确认: raw_has_lead → has_lead → acc_has_lead       │
│     │   ├── 弯道记忆保活                                             │
│     │   └── ACC释放条件：偏出车道/远离/速度骤降                     │
│     ├── CurveHoldManager.update() → in_curve_hold                  │
│     ├── AebAlertManager.update() → alert_state                     │
│     ├── compute_longitudinal_policy() → LongitudinalContext        │
│     │   ├── ACC跟车模式 (有前车): D+I+V+A控制器                     │
│     │   ├── 弯道保持模式 (弯道中丢失前车): PI速度保持                │
│     │   ├── 巡航模式 (无前车无弯道): 定速                             │
│     │   ├── AEB制动叠加                                             │
│     │   └── 边界制动取大: max(lon_cmd, boundary_brake)              │
│     └── LonSmoothing.update() + limit_non_aeb_lon()               │
│         ├── 一阶低通平滑（带变化率限幅）                              │
│         ├── 接管保护: 更严的max_rate_override                       │
│         └── 非AEB纵向指令限幅（弯道减速等）                          │
│                                                                     │
│  ⑧ NaN防护: _find_nonfinite_tx_fields()                           │
│     └── 任一字段NaN/Inf → 整帧替换为安全值(δ=0, 轻制动); return   │
│                                                                     │
│  ⑨ clamp + 接管保护限幅                                             │
│     └── _apply_takeover_guard(): 窗口内对delta/lon更严限幅          │
│                                                                     │
│  ⑩ 输出                                                            │
│     ├── _publish_outputs() → 8个ROS话题                             │
│     ├── esp32.send() → 串口控制帧 (CRC8校验)                        │
│     ├── _sync_peer_output() → UDP心跳广播                           │
│     ├── _log_cycle_summary() → 限频日志                             │
│     └── _record_telemetry() → 非阻塞CSV投递                         │
│                                                                     │
│  ⑪ cycle_count += 1                                                │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.2 横向控制（LKA）详解

```
输入: road_psi, ego_yaw, lane_offset, ego_v, filtered_curv

    ┌───────────────────┐
    │  道路航向低通滤波  │ ← ROAD_PSI_FILTER_ALPHA
    │  rrate = Δψ/Δt    │
    └────────┬──────────┘
             │
    ┌────────▼──────────┐
    │  预览航向计算      │ ← adaptive_preview_time(v, curv)
    │  ψ_preview = ψ_road + t_preview * ω_road + k_preview * CTE  │
    └────────┬──────────┘
             │
    ┌────────▼──────────┐
    │  航向PID控制       │
    │  err = ψ_preview - ψ_ego                                    │
    │  P项: K_PSI_P * err                                         │
    │  I项: K_PSI_I * ∫err (低速衰减, 饱和衰减)                    │
    │  D项: K_PSI_D * Δψ_rate (rrate)                             │
    │  → upd_psi                                                    │
    └────────┬──────────┘
             │
    ┌────────▼──────────┐
    │  曲率前馈          │
    │  δ_ff = K_FF * κ * (L + t_preview * v) * STEER_SIGN          │
    │  弯道CTE增益加成 / 前馈曲率衰减                               │
    └────────┬──────────┘
             │
    ┌────────▼──────────┐
    │  CTE修正            │
    │  filtered_cte ← 低通(lane_offset)                             │
    │  δ_cte = K_CTE * filtered_cte + K_CTE_D * d(filtered_cte)/dt │
    └────────┬──────────┘
             │
    ┌────────▼──────────┐
    │  边界修正           │
    │  软约束: 指数衰减修正 (lane_warn_margin外)                     │
    │  硬约束: 线性强修正 (lane_hard_margin外)                        │
    │  制动: boundary_brake (最大纵向减速度)                         │
    │  独立于道路帧门控——按最新lane_offset刷新                       │
    └────────┬──────────┘
             │
    ┌────────▼──────────┐
    │  合成 + rate limit │
    │  δ_raw = upd_psi + δ_cte + δ_ff                              │
    │  δ += boundary_delta (独立rate limit)                          │
    │  δ_final = rate_limit(δ, last_delta, MAX_DELTA_RATE * dt)    │
    └───────────────────┘
```

## 五、主备切换机制

### 5.1 心跳协议

```
主机 → 备机: "HB:1 SEQ:N PSI:xx DELTA:xx ACC:xx\n"  (每100ms)
备机 → 主机: "BACKUP:1 ACTIVE:0/1\n"                (每100ms)
```

**心跳格式说明**：
- `SEQ`: 单调递增序号，用于检测主机进程卡死（消息源被缓冲区冻结）
- `PSI/DELTA/ACC`: 主机当前控制输出，作为备机接管时的种子

### 5.2 切换流程

```
时间线:
────────────────────────────────────────────────────────────►

主机正常运行:    HB:1 SEQ:1 | HB:1 SEQ:2 | HB:1 SEQ:3 | ...
                  ↓ 复制    ↓ 复制       ↓ 复制

主机故障:        ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ (沉默 OR SEQ停滞)

备机检测:         .....正常..... | 超时判定(>0.5s) | ← False→True 边沿
                                                        │
                                          接管初始化:     │
                                          lon_smooth.reset(lon_seed)
                                          last_delta = delta_seed
                                          启动保护窗口
                                                        │
备机输出控制:                                  ┌────────┴────────┐
                                              │ 接管保护窗口内    │
                                              │ delta/lon 更严    │
                                              │ 变化率限制         │
                                              │ TAKEOVER_DELTA_RATE│
                                              │ TAKEOVER_LON_RATE  │
                                              └────────┬────────┘
                                                       │
主机恢复:                    ← HB:1 SEQ:>N ←  ─ ─ ─ ─ ─┤
                                                        │
                                          备机退回:      │
                                          保持active      │
                                          HB_STANDBY_HANDOFF_S│
                                          后切回standby   │
```

### 5.3 Flapping 保护

当主机时断时续（flapping）时：
- 距上次接管不到 `TAKEOVER_COOLDOWN_S` 的接管只延长保护窗口，不重置控制器状态
- 避免反复从主机种子覆盖内部状态，导致控制器振荡

| DIST | 前车距离(m) | 0-999.99 |

| OFFSET | 车道偏移(m) | |
| LEADV | 前车投影速度(m/s) | |
| DSAFE | 最小安全距离(m) | |
| WMRN | 预警余量(m) | |
| WHRD | 硬边界余量(m) | |
| CURV | 滤波曲率(1/m) | |
| CRC | CRC-8/MAXIM校验 | Dallas多项式0x31 |

## 七、安全机制

### 7.1 三层安全防护

```
┌─────────────────────────────────────────────────────┐
│                 第一层：NaN/Inf 防护                  │
│  _find_nonfinite_tx_fields() 在clamp之前检查          │
│  任一字段NaN/Inf → 整帧替换为安全回退                 │
│  (δ=0, 轻制动, 全部硬编码有限值)                      │
├─────────────────────────────────────────────────────┤
│                 第二层：连续异常计数                   │
│  CTRL_CONSECUTIVE_ERROR_LIMIT 次连续异常              │
│  → _send_emergency_stop() (最大制动 + 方向盘归零)    │
│  限频100ms/次                                       │
├─────────────────────────────────────────────────────┤
│                 第三层：感知中断制动                   │
│  ego/road 卡帧或未就绪 → 无法正常解算                 │
│  超过 SENSOR_TIMEOUT_BRAKE_S → 轻制动                │
│  (防止车辆保持上一帧加速指令)                         │
└─────────────────────────────────────────────────────┘
```

### 7.2 CRC 帧校验

Jetson→ESP32 每帧附带 CRC-8/MAXIM（Dallas 多项式 0x31），ESP32 侧验证不匹配则丢弃。防止 UART 线路噪声导致的单字节翻转污染控制量。

### 7.3 卡帧检测

`evaluate_control_health()` 不仅检查话题是否收到，还用 `now - last_rx > SENSOR_STALE_TIMEOUT_S` 检测卡帧（话题已收到但停更）。卡帧时视为不就绪，走轻制动降级路径。

## 八、实时性设计

### 8.1 线程模型

```
┌─────────────────────────────────────────────────────────┐
│                    主控制线程 (100Hz)                    │
│  定时器回调 → _control_loop_impl()                      │
│  只做非阻塞操作: 读写property/队列put_nowait/原子变量    │
├──────────────┬───────────────┬──────────────────────────┤
│  esp32-tx    │  esp32-rx     │  UDP心跳线程 ×N           │
│  daemon      │  daemon        │  (发送+接收各1)          │
│  从队列取帧   │  持续读串口    │  周期发送/接收           │
│  写入串口    │  解析回读值    │  更新共享状态            │
│  断线重连    │  断线重连      │  检测超时/序号停滞       │
├──────────────┴───────────────┴──────────────────────────┤
│               log QueueListener 线程                    │
│               取队列 → 写文件 (RotatingFileHandler)     │
├─────────────────────────────────────────────────────────┤
│               telemetry CSV 写盘线程                    │
│               取队列 → csv.writerow → 周期flush         │
└─────────────────────────────────────────────────────────┘
```

**关键原则**：主线程的所有阻塞调用（串口读/写、日志写盘、CSV落盘）都由后台线程承担。主线程只做 `put_nowait` 和原子 property 读取。

### 8.2 丢帧策略

- ESP32 发送：容量4的丢旧保新队列，控制帧只关心最新值
- 遥测写盘：容量4096的队列，满时丢最旧帧并计数
- 日志：QueueHandler→无限队列→QueueListener线程，避免阻塞

### 8.3 日志限频

- `LOG_EVERY_N_CYCLES = 100`：每100个周期输出一次摘要
- `RateLimitedCritical`：同类CRITICAL事件首次全量，1s内降为WARNING
- NaN防护日志：1s窗口聚合统计

## 十一、设计亮点与潜在关注点

### 11.1 设计亮点

| 亮点 | 说明 |
|------|------|
| **纯计算管线可离线驱动** | `pipeline.py` 抽取使算法与I/O彻底解耦，支持CSV回放回归对比 |
| **三层安全防护** | NaN检测→连续异常停车→感知中断制动，覆盖软件异常到硬件故障 |
| **CRC帧校验** | 防止UART噪声导致控制量翻转 |
| **丢旧保新队列** | 控制帧只关心最新值，4帧缓冲足够 |
| **卡帧检测** | 不仅检测"未收到"，还检测"收到过但停更" |
| **接管保护+flapping冷却** | 避免非物理跳变和反复初始化 |
| **日志限频+异步落盘** | QueueHandler保证不阻塞控制线程 |
| **MPC可选替换** | drop-in接口，运行时无需修改其他代码 |
| **Python 3.6兼容** | JetPack 4.x约束下的工程实践 |

### 11.2 潜在关注点

| 关注点 | 说明 | 严重性 |
|--------|------|--------|
| **`from config import *` 星导入** | 所有模块星导入config，命名空间污染风险，参数来源不透明 | 中 |
| **VehicleSignals 可变性** | 回调写入+控制环读取的无锁共享对象，Python GIL保证原子性但语义上令人不安 | 低 |
| **串口断线重连** | 后台线程自动reopen，但reopen期间可能丢帧，ESP32侧无帧序号检测重连后的帧丢失 | 中 |
| **LaneWidthEstimator冷启动** | 初始使用LANE_WIDTH_MIN偏保守，但样本不足期间的边界余量可能偏紧 | 低 |
| **超车状态机与双车道强耦合** | OVT_LANE_OFFSET_M 硬编码为左车道偏移，道路模型变更需重写 | 中 |
| **config.py 集中管理但缺少运行时校验** | 参数间约束关系（如 ACC_KD * ACC_TIME_GAP 的安全范围）没有代码级断言 | 低 |
| **serial_link.py rx线程解析** | `parse_tagged_lines` 按标签解析回读，标签不匹配静默跳过无诊断 | 低 |
| **replay.py 仿真模型简陋** | 单质点纵向+运动学横向，不替代Simulink HIL，仅用于算法回归 | 信息 |

## 十二、总结

本系统是一个**工业级嵌入式ADAS控制节点**，在Jetson Nano有限算力下实现了完整的LKA+ACC+AEB功能链。核心设计围绕**实时性**（10ms预算）和**安全性**（三层防护+CRC校验），通过后台线程卸载所有阻塞I/O，保证100Hz控制周期不被拖慢。`pipeline.py`的纯计算抽取使得算法可离线回归验证，是整个工程质量的重要保障。

主备双机热备设计周全：序号停滞检测、接管种子初始化、保护窗口限幅、flapping冷却，覆盖了实际部署中最常见的故障模式。

代码注释和日志均为中文，与团队语言一致，便于维护。
